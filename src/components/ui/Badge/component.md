# Badge 组件说明

## 1. 组件定位
`Badge` 组件用于展示状态、分类标签、提示角标等小型元数据文本。在业务中常用于在项目卡片、模板卡片上标识“NEW”或者项目的特定垂类（如 `3D PIPING`）。

## 2. 搭配方式与应用场景
- **结合卡片使用**：放置在卡片的顶角处。
- **结合列表使用**：在表格或列表中作为状态列突出优先级或分类。

## 3. 可修改的范围参数 (Props)
- `children` (ReactNode): 标签内文字。需尽量简短（不建议超过 12 个字母）。
- `variant` ('solid' | 'outline' | 'pill'):
  - `pill` (默认): 白底胶囊状，搭配较高的轻薄阴影，常用于带有渐变背景的卡片上方。
  - `solid`: 柔和的半透明/纯色底色，用于高对比卡片内的醒目位置（如蓝色背景上的 `bg-white/20`）。
  - `outline`: 边框透明底色，适用于极简数据展示。
- `colorClassName` (string): 文字颜色专用的 Tailwind Class。例如 `text-[#0048e1]`，用以控制具体的主题色。
- `className` (string): 容器的额外扩展样式类（可用于布局位置微调，例如 `ml-2`）。

## 4. 设计规范禁忌（不建议修改）
- **绝不允许随意修改圆角和字体粗细**：全局设计规范要求使用 `uppercase tracking-widest font-bold`，不能随意修改成正常大小写或细字体以保障 UI 边缘的锐利视觉体验。
- **尽量不干预内边距 padding**：组件已预设好 `px-3 py-1` 及基于断点的 `text-[10px] sm:text-xs` 自适应文本大小引擎。
