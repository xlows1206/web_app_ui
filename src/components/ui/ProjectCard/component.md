# ProjectCard 组件说明

## 1. 组件定位
`ProjectCard` 高阶功能名片是 Workspace 主页等页面下用来呈现用户关键开发项目节点的标准展示框体。依靠将高度复杂的样式及炫光结构执行彻底内聚封装，此组件屏蔽并在前端剥离了成百上千行的脏代码和 Tailwind 类声明引擎背景流。

## 2. 交互与特性
- 组件已经将 `Editorial Pulse` 卡片风格里的“微发光体背景效果”提取并应用。且按照严格相等比例的相同空间透湿度缩小到了组件内（并在内部采用了软化、削弱过对比度的底层深蓝色呼吸脉动光晕）。
- 当系统或用户引发 `hover` 时，这组位于背景底层的高级呼吸炫光将会非常顺滑并且富有节律地激活扩散，但是这绝对不伴随令人反胃的 `hover-scale` 抖动缩放位移。
- 在视图实现层面，其内部高度集成了 UI 通用子组件栈： `<AvatarGroup />`、`<Badge />` 和 `<IconButton />` ，完美展示原子组件互相镶嵌使用的规范与魅力。

## 3. 可修改的范围参数 (Props)
- `title` (string): 主级大字标题，代码层面预配了 `line-clamp-2` 能够做到坚如磐石的防超长溢出文本阶段处理。
- `tags` (ProjectTag[]): 包含了标签标识名及 Tailwind 专供类的 `colorClass` 数据阵型接口。
- `updatedAt` (string): 浮于底端部右侧的时效追踪类文本字符串。
- `avatars` (string[]): 被底端叠加层组件直接穿透消费的人员标识色组参数栈。
- `extraCount` (number): 超发隐藏的非活跃参与人员数值计量。
- `className` (string): 用于让父容器通过 flex 或者 grid 根据需要控制调节卡片留白等的外部保留覆盖接口口属性。

## 4. 设计规范禁忌（不建议修改）
- **发光渐变光污染层管控**：其内部自带的光源投射角度、透明度、发散距离算法及软化叠加逻辑混杂 (如 `primary/[0.06]`, `blur-2xl`) 是历经常规对比才针对严格的小卡片尺寸进行特调的黄金弱化比例模型。外部组件如果没有十分强劲的视觉改版需求，**切忌通过注入子层强制调大并弄脏覆盖这套背景发光幕系统** ！
- **卡片边缘下压形变骨架模型约束**：在任何业务上下文中，不可随意剥夺覆写由 `active:scale-[0.99]` 和外拓阴影动画带来的极佳干练下压物理手感响应深度。
