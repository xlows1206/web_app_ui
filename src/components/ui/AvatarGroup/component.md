# AvatarGroup 组件说明

## 1. 组件定位
`AvatarGroup` 用于显示项目卡片内部或协作列表视图里的多个人员参与成员。它高度封装且自动处理了 CSS 的图像覆盖堆叠问题（基于递归减法的 `z-index` 及负边距 `-ml-3` 叠层效果）。

## 2. 交互与特性
- 将提供的代表头像颜色的数组（或者未来的真实图片 URL）按照数组逻辑逐层紧密重叠渲染。
- 支持在末尾渲染 `+N` 的额外人员计数。该气泡会自动处于所在最底层显示。

## 3. 可修改的范围参数 (Props)
- `avatars` (string[]): 代表用户头像颜色类或者图片引用的渲染数组。
- `extraCount` (number, 可选): 假如包含更多未展示组员，传入数字将在右侧气泡显示对应 `+N`。
- `avatarSizeClass` (string): 用于覆盖默认宽高的尺寸。默认提供 `w-9 h-9`，适用于小卡片，如果用遇到首页大号组件场景可以传入 `w-12 h-12`。
- `borderColorClass` (string): 用来处理由于负叠层导致的头像之间需要“镂空”切割视觉的覆盖假边框色。默认为父级的表面色 `border-surface-container-low`。

## 4. 设计规范禁忌（不建议修改）
- **必须不要直接插手修改深层的 Z-Index**：Z层级深度逻辑已经过内置硬编码的 `30 - idx` 计算排版保障，防止人员渲染过多导致组件直接因层级穿透被外部控件错误覆盖。
- **不可在外部剥离 `-ml-3` 参数**：该核心类实现了头像叠起串联的标志性视觉重叠，剥离后成员头像将变得零碎割裂，不再属于同一团队隐喻。
